# Podcast转录工具

一个支持本地文件和在线链接的Podcast转录桌面应用，能够自动区分主持人与嘉宾，并在此基础上构建自我强化学习平台。

## 核心目标

本项目旨在通过Podcast内容分析，探索以下核心问题：

1. **成功与失败的路径研究**：
   - 成功的路径多还是失败的路径多？
   - 成功的路径是否相似，还是失败的路径更具相似性？
   - 如果成功有迹可循，如何通过学习成功人士的经验（评论、Podcast等）提升个人能力？
   - 如果成功没有固定模式，如何应对不确定性和变化？

2. **普通人的成长与发展**：
   - 如果成功不是一个状态或结果，而是一种趋势，普通人该如何发掘增量？
   - 如何识别并驾驭个人成长的走向？
   - 如何在复杂多变的环境中保持持续进步？
   - 如何将学习成果转化为实际行动和价值？

3. **自我强化学习机制**：
   - 构建基于Podcast内容的个人学习闭环
   - 实现从信息获取到行动执行的完整转化
   - 持续优化学习模型和分析能力
   - 最终实现计算成本大于通信成本的高效学习

## 功能特性

- 📁 **支持本地文件**：MP3、MP4、M4A、WAV、AAC格式
- 🔗 **支持在线链接**：直接输入Podcast URL进行转录
- 🎤 **自动区分发言人**：识别主持人与嘉宾
- 💻 **跨平台支持**：Windows、macOS、Linux
- 📱 **移动端扩展准备**：保留iOS和Android适配可能性
- 🧠 **自我强化学习**：基于转录内容构建个人学习平台

## 技术栈

- **主框架**：Electron
- **前端**：HTML5 + CSS3 + JavaScript
- **后端服务**：Node.js + FastAPI
- **语音识别**：FunASR本地模型 + 国内API接口（百度AI、阿里AI、腾讯AI等）
- **网络请求**：Axios
- **AI框架**：PyTorch + Transformers
- **音频处理**：FFmpeg

## 安装与运行

### 环境要求

- Node.js 18.x 或更高版本
- Python 3.10 或更高版本
- npm 或 yarn
- Conda（推荐，用于管理Python环境）

### 安装步骤

1. 克隆或下载项目到本地

2. 安装前端依赖
   ```bash
   npm install
   ```
   
   如果遇到依赖冲突，可以尝试：
   ```bash
   npm install --legacy-peer-deps
   ```

3. 配置并启动后端服务
   ```bash
   # 进入后端目录
   cd backend
   
   # 创建conda环境（推荐）
   conda create -n podcast-transcriber python=3.10 -y
   conda activate podcast-transcriber
   
   # 安装后端依赖
   pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt
   
   # 启动后端服务
   python run_server.py
   ```

4. 运行前端应用
   ```bash
   npm start
   ```

### 构建应用

```bash
npm run build
```

## 使用说明

1. **选择本地文件**：点击"选择本地文件"按钮，选择您的Podcast文件
2. **或输入在线链接**：在输入框中粘贴Podcast的在线URL
3. **开始转录**：点击"开始转录"按钮
4. **查看结果**：转录完成后，会显示带有发言人标记的文字稿
5. **使用学习平台**：基于转录内容进行自我强化学习

## 核心功能说明

### 1. 文件处理流程

```
本地文件/在线URL → 下载（如果是URL）→ 格式转换为WAV → 语音识别 → 区分发言人 → 显示结果
```

### 2. 发言人区分

应用使用语音识别模型的说话人分离功能，自动区分主持人与嘉宾。支持本地FunASR模型和国内云服务API。

### 3. 自我强化学习平台

基于Podcast转录内容，构建个人学习平台，具备以下特性：

#### 🎯 核心功能

1. **角色扮演分析**：分析当前系统中的生态位，理解各参与者角色和关系，评估自身定位
2. **未来预测**：基于转录内容预测未来趋势和潜在后果，提供前瞻性洞察
3. **非共识观点识别**：发现隐藏的非共识观点，识别打破常规的思考角度
4. **优势与增量分析**：
   - 优势分析：识别超越平均水平的核心竞争力
   - 增量分析：发现个人成长和进步的关键点
5. **可视化展示**：
   - 通过交互式画布展示复杂关系
   - 使用动态表格呈现结构化数据
   - 支持多种图表类型（柱状图、折线图、网络图等）
6. **可行动建议**：生成具体、可执行的行动方案，包含时间线和优先级
7. **多角度提问**：从5个不同视角提出关键问题，拓展思考维度：
   - 客户视角
   - 竞争对手视角
   - 投资者视角
   - 行业专家视角
   - 未来用户视角
8. **市场基准测试**：
   - 基于金融市场数据评估产品/服务效果
   - 结合Polymarket等预测市场进行验证
   - 提供相对竞争力排名
9. **捕捉市场趋势**：识别市场风向和机会窗口，把握时机
10. **学术论文扩展**：
    - 将分析扩展到学术层面
    - 关联相关研究领域
    - 生成学术风格的分析报告
11. **再生Podcast**：
    - 基于原内容创建改进版本
    - 形成内容迭代循环
    - 持续优化内容质量
12. **计算成本优化**：
    - 目标：计算成本大于通信成本
    - 优化模型推理效率
    - 实现资源智能分配
13. **失败案例分析**：
    - 收集并分析常见失败案例
    - 从反面角度提供见解
    - 生成风险规避策略
14. **事实与观点区分**：
    - 自动识别内容中的事实陈述
    - 区分主观观点和客观事实
    - 提供证据支持分析

#### 🔄 学习闭环机制

```
Podcast内容 → 转录与分析 → 学习平台处理 → 生成见解与建议 → 行动执行 → 反馈收集 → 模型优化 → 新一轮分析
```

## 产品需求文档 (PRD)

### 产品概述

构建一个基于Podcast转录的自我强化学习平台，通过分析大量音频内容，探索成功与失败的路径规律，帮助用户：
- 从音频内容中提取有价值的见解
- 发掘个人成长的增量机会
- 驾驭发展趋势
- 实现持续进步和自我提升
- 最终形成计算成本大于通信成本的高效学习闭环

本平台将通过深入研究成功人士的经验和失败案例，为普通人提供可行动的成长策略和发展路径。

### 用户痛点

1. 大量音频内容难以高效利用
2. 缺乏系统的方法从内容中提取价值
3. 难以发现非共识观点和隐藏机会
4. 需要具体的行动建议而非泛泛而谈
5. 缺乏多角度思考的工具
6. 现有学习内容过于注重结果，缺乏思考过程的展示
7. 学习过程中难以获得及时正反馈，缺乏成就感
8. 无法有效模仿成功者的思考方法和思路

### 核心用户场景

1. **内容创作者**：分析竞品Podcast，发现市场机会，生成改进建议
2. **投资者**：从行业Podcast中识别趋势，评估项目潜力
3. **职场人士**：从专业Podcast中学习，生成可行动的职业发展建议
4. **研究者**：将Podcast内容转化为学术分析材料
5. **普通学习者**：系统化学习感兴趣领域的内容

## 风险与挑战

1. **模型性能**：本地模型可能在复杂场景下表现不佳
2. **计算资源**：大规模模型推理需要大量计算资源
3. **数据隐私**：用户数据安全和隐私保护
4. **内容质量**：低质量Podcast可能影响分析结果
5. **用户 adoption**：新学习方式的接受度
6. **技术复杂度**：多模块集成的复杂性

## 风险缓解策略

1. **模型优化**：
   - 实现模型自动选择机制
   - 开发模型蒸馏技术

## 配置说明

### 语音识别配置

后端服务支持本地FunASR模型和国内云服务API，可在`backend/main.py`中配置：

```python
# 本地FunASR模型配置
model_dir = "FunAudioLLM/Fun-ASR-Nano-2512"

# 或配置国内云服务API
# ...
```

### 环境变量配置

在`backend/.env`文件中配置API密钥：

```
# 百度AI语音识别API配置
BAIDU_API_KEY=your_api_key_here
BAIDU_SECRET_KEY=your_secret_key_here
```

### FFmpeg配置

需要在系统中安装FFmpeg：

- **Windows**：下载FFmpeg并添加到环境变量
- **macOS**：`brew install ffmpeg`
- **Linux**：`sudo apt install ffmpeg`

2. 首次运行后端服务时，会下载FunASR模型，需要网络连接
3. 大文件转录可能需要较长时间
4. 确保系统已安装FFmpeg
